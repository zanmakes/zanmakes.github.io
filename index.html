  1. Creating the User Entity

    This entity is the most important one because others will depend on this a lot. It's a Strong Entity because it doesn't need another entity to depend on it.

    Attributes

    id
    Uniquely identifies entity.
    full_name
    Uniquely identifies entity. It's a multivalued attribute because it can be split into two more attributes -- first_name and the last_name attributes.

    Relationships

    • Enrollments - A user that's a student can enroll in a course. Zero to Many.
    • Course - Determines whether or not an instructor has a course they are managing. Zero to One.
    • Student Information - A user that's a student has data here exclusive to a student. Zero to One
    • Assignment - An instructor of a course can create, read update, and delete a course's assignment. One to Many.
    • Assignment Submissions - A student of a course can submit a course's assignment. Many to Many.

  2. Creating the Student Information Entity

    This distiguishes a student and instructor from the user. If the user does not have a relationship here, it means they are an instructor, otherwise, they are a student. This is a Weak Entity because this Entity's existense depends on the User Entity.

    Attributes

    id
    Uniquely identifies entity.
    user_id
    Identifies which user is a student.
    final_grade
    Student's final grade.

    Relationships

    • User - information exclusive to students. Zero to One.

  3. Creating the Course Entity

    The Courses that Users can create, read, update, and delete. This is a Weak Entity because this Entity's existense depends on the User Entity.

    It could have been a Strong Entity but that would have made the ER diagram more complicated.

    Attributes

    id
    Uniquely identifies entity.
    title
    Name of the course.
    instructor_id
    Refers to the instructor responsible of managing the course.

    Relationships

    • User - links to instructor responsible for the management of course. For simplicity, it's a One to One relationship.
    • Enrollment - links to student that enrolled in a course to determine whether or not the assignment submission is applicable to them. For simplicity, it's a One to One relationship.

  4. Creating the Enrollments Entity

    Lists students enrolled in a course. This is a Weak Entity because this Entity's existense depends on the User, and Course Entities.

    Attributes

    id
    Uniquely identifies entity.
    student_id
    Identifies which user is a student.
    course_id
    Student's final grade. It's a Multivalued key because a student can be enrolled in multiple courses.

    Relationships

    • User - information exclusive to students. One to One.
    • Course - Identifies what course the student is enrolled in. Zero to Many.

  5. Creating the Assignments Entity

    Management of Assignments. This is a Weak Entity because this Entity's existense depends on the User, and Course Entities.

    Attributes

    id
    Uniquely identifies entity.
    assignment_title
    Name of the assignment.
    course_id
    The course this assignment is from.

    Relationships

    • User - information exclusive to students. One to One.
    • Course - Identifies what course the student is enrolled in. Many to Many.
    • Assignment Submission - Gets information of student's submission Zero to Many.

  6. Creating the Assignment Submissions Entity

    Management of Assignments' Submissions. This is a Weak Entity because this Entity's existense depends on the User, Course, and Assignment Entities.

    Attributes

    id
    Uniquely identifies entity.
    course_id
    Identifies which course this assignment is from.
    assignment_id
    Identifies the assignment.
    student_id
    Identifies the student that submitted the assignment.
    file_location
    The file to submit.

    Relationships

    • User - information exclusive to students. One to One.
    • Course - Identifies what course the student is enrolled in. Many to Many.
    • Assignment - Identifies what assignment the student is submitting in. One to Many.
